# Домашнее задание: Университетская система

## Описание задачи

Твоя задача — спроектировать и реализовать базовую консольную систему для управления университетом.

Система должна позволять создавать и управлять объектами, представляющими студентов, преподавателей, группы, предметы (пары) и аудитории.

---

## Основная функциональность

### Модели (`Classes`)

Спроектируй и реализуй следующие классы:

#### `Student`
Поля:
- имя
- возраст
- номер студенческого билета

#### `Teacher`
Поля:
- имя
- возраст
- предмет, который преподает

#### `Group`
Поля:
- название группы
- список студентов

#### `Subject`
Поля:
- название предмета
- преподаватель
- аудитория

#### `Auditorium`
Поля:
- номер кабинета
- вместимость

---

## Связи между классами

- `Student` принадлежит одной `Group`
- `Group` имеет список `Student` и список `Subject`
- `Subject` имеет одного `Teacher` и одну `Auditorium`

---

## Консольное меню

Реализуй простое консольное меню для взаимодействия с системой.

Меню должно позволять:

1. Создавать новые объекты (студента, преподавателя и т.д.)
2. Добавлять студента в группу
3. Добавлять предмет в группу
4. Выводить информацию:
   - "Все студенты в группе X"
   - "Все предметы в группе Y"
   - "Преподаватель Z и его предметы"

# Архитектура и паттерны проектирования

## Принципы SOLID

### Single Responsibility Principle (SRP)

Раздели логику создания, хранения и отображения данных по разным классам.

Например:
- `Service`
- `Repository`
- `ConsoleUI`

### Dependency Inversion Principle (DIP)

Примени принцип, если это уместно.

Например, если в будущем планируется хранение данных не в памяти, а в базе данных.

---

## Creational Patterns

### Singleton

Используй этот паттерн для класса, который управляет всей системой (например, `UniversityManager`).

Убедись, что можно создать только один экземпляр этого класса.

### Factory Method

Используй паттерн "Фабричный метод" для создания экземпляров разных типов.

Например:
- `StudentFactory`

---

## Structural Patterns

### Composite

Используй паттерн "Компоновщик" для представления иерархии.

Например:
- `Group` может содержать `Student`

---
